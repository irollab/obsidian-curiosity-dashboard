# Curiosity Dashboard 数据格式

插件以 Vault 中的 Markdown 和 YAML Frontmatter 为唯一事实来源。本页列出 V1 实际读取的数据合同。

## 选题字段

| 字段 | 类型 | 用途 | 缺失或无效时 |
| --- | --- | --- | --- |
| `type` | 字符串 | 必须为 `选题` 才会被收录 | 忽略该文件 |
| `title` | 非空字符串 | 显示标题 | 从文件名去掉开头期数后取值 |
| `status` | 字符串 | `已立项` 进入制作队列 | 按空字符串处理 |
| `issue` | 正安全整数 | 期数和关联键 | 从文件名开头的数字读取；仍不可用则忽略该文件 |
| `homepage_focus` | 布尔值 | 严格等于 `true` 时成为当前作品 | 非当前作品 |
| `stage` | 字符串 | 内容生产阶段 | 显示“未知阶段”并禁用推进 |
| `next_action` | 非空字符串 | Hero 的下一步 | 显示“下一步未设置” |
| `due_date` | 非空字符串，期望为 `YYYY-MM-DD` | 本周排期和队列截止日 | 缺失时不进入本周列表且队列显示“未设置”；非法日期不进入本周列表，队列仍显示原文便于修正 |
| `priority` | 非空字符串 | 制作队列优先级 | 显示“未设置” |
| `script_path` | Vault 内路径字符串 | 明确关联脚本 Markdown | 在脚本目录按期数查找唯一 Markdown 候选 |
| `asset_path` | Vault 内路径字符串 | 明确关联素材文件或目录 | 在素材目录按期数查找唯一文件或目录候选 |
| `review_path` | Vault 内路径字符串 | 明确关联复盘 Markdown | 先在复盘目录按期数查找唯一 Markdown 候选；仍无关联时才进入最新复盘选择 |

`channel`、`platform`、`created` 等旧字段不会被删除或改写，但 V1 的选题查询不使用它们推断上述字段。

## 复盘 Frontmatter 字段

| 字段 | 类型 | 用途 | 缺失或无效时 |
| --- | --- | --- | --- |
| `created` | 严格 ISO 日期或日期时间字符串 | 在未有明确复盘关联时排序 | 尝试 `publish_date` |
| `publish_date` | 严格 ISO 日期或日期时间字符串 | `created` 无合法值时的排序 fallback | 该复盘不参与“最新复盘”自动选择 |

复盘的 `type`、`status`、`tags` 可供 Vault 本身组织和搜索，但 V1 的复盘解析器不使用它们筛选文件或推断指标。明确的 `review_path` 只要是安全、存在的 Markdown 即可读取；自动选择仅在配置复盘目录内进行。

## 阶段与焦点约束

五个合法阶段的顺序固定为：

`选题 → 策划 → 制作 → 发布 → 复盘`

- 推进一次只前进一阶段，必须经过确认，`复盘` 是终止状态。
- Vault 中必须恰好有一篇 `type: 选题` 文件设置 `homepage_focus: true`。
- 零个焦点显示设置引导；多个焦点显示冲突列表，插件不会自动选择。

## Checklist

任务只从当前选题文件中下列精确二级标题下读取：

```markdown
## 本期执行清单
- [x] 已完成任务
- [ ] 未完成任务
```

标题文字、级别不匹配，或 Checkbox 缺少任务文字时，不会读取。写入前会重新解析最新文件；行号、文字或勾选状态已变化时中止，不会改动其他任务。

## 复盘表格

### 跨平台表

表头必须同时含“平台”和一个支持的播放列。示例：

```markdown
| 平台 | 采集时间 | 播放/观看 | 点赞 | 收藏 | 评论 | 分享 |
| --- | --- | ---: | ---: | ---: | ---: | ---: |
| 平台A | 2026-06-22 | 100 | 10 | 8 | 2 | 1 |
```

播放列别名：`播放/观看`、`播放/阅读`、`播放`、`观看`。分享列别名：`分享`、`转发`、`分享/转发`。其他支持列为 `采集时间`、`点赞`、`收藏`、`评论`。

### 单平台数据快照

表头不含“平台”时，必须含 `时间点` 或 `采集时间`、播放列，且附近可唯一确定 `平台：……`。解析最后一条有指标的快照。

```markdown
平台：示例平台

| 时间点 | 播放 | 点赞 |
| --- | ---: | ---: |
| 24h | 100 | 10 |
```

表格仅显示实际存在的字符串值；不转换平台口径，不计算、不补齐、不推断缺失数据。找到多张表时使用第一张可解析且有指标的表。

## 复盘日期与关联规则

1. 当前选题的安全、存在且以 `.md` 结尾的 `review_path` 优先，它可位于自定义目录。
2. `review_path` 缺失时，关联器在配置的复盘目录内按 `issue` 匹配 Markdown；只有唯一候选时自动关联，多个候选必须由用户选择并写入明确路径。
3. 没有明确关联时，最新复盘只在配置的复盘目录内选择：先读 `created`，无合法值再读 `publish_date`，支持严格 ISO 日期/时间。
4. 两个日期都缺失或无效时，不使用文件修改时间、文件名或正文猜测。

## 评论需求

读取二级或更深的精确标题 `评论区需求`、`评论反馈` 或 `评论样本` 下的列表文字，直到同级或更高级标题。代码块、HTML 注释和无法验证的正文不会当作评论。没有原文时显示“暂无可验证评论内容”；搜索词等旁证应在文档中明确标注为旁证。

## 错误、空状态与移动端

- 背景为空或文件不存在：Hero 使用纯黑背景。
- 无本周作品、无制作队列、无平台数据：显示对应空状态，不伪造占位数字。
- 复盘表无法解析：保留本地复盘入口和解析说明。
- 关联多候选：显示候选并要求选择，不自动猜测。
- 模板不存在：不创建空壳文件，返回设置修正路径。
- 写入期间源内容变化：中止并提示刷新后重试。
- 移动端始终只读；即使启用简化移动视图，也不允许勾选任务、推进阶段、修改关联或从模板创建文件。
