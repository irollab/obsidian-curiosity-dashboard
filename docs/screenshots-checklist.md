# 截图清单（手动补充）

README 引用 `docs/screenshots/` 下的截图。Obsidian 是桌面应用，需手动截图后放入此目录。

## 截图方法

系统截图工具：`Win + Shift + S`（Windows）框选区域 → 自动进剪贴板 → 粘贴到画图或直接保存为 PNG 到 `docs/screenshots/`。

推荐用 **1280×800** 左右的宽屏比例，裁掉 Obsidian 顶部的文件标签栏（只保留插件面板）。

## 建议的 4 张截图

| 文件名 | 内容 | 重点展示 |
|---|---|---|
| `overview.png` | 「概览」tab | Hero 大图焦点选题 + 焦点切换器 + 本周排期 + 制作队列 |
| `discover.png` | 「发现」tab | 热点列表（多来源彩色标签 + 分类过滤 chip）+ 受众信号 + 翻页 |
| `workflow.png` | 「工作流」tab | 按阶段分组的 Codex 提示词卡片 + 复制按钮 |
| `data.png` | 「数据」tab | 渠道脉搏表格（平台彩色标签 + 各指标列分色 + 高值放大突出） |

放齐这 4 张后，README 的截图区会自动显示。

## 脱敏提示（重要）

截图会进入公开仓库，**务必先处理敏感信息**：

- **选题标题/期数**：换成虚构的示例标题（如「给 Codex 的任务说明模板」「Obsidian+Codex 搭建选题系统」），不要暴露未发布的真实选题。
- **复盘数据**：播放量/点赞等可换成整数示例（如 `1200`、`88`），不要暴露真实账号数据。
- **文件路径**：检查 Hero 里的路径是否含个人标识。
- **评论区内容**：「数据」tab 的评论样本要用虚构示例，不能是真实用户评论。
- **账号信息**：裁掉 Obsidian 左侧文件树（可能含你的目录结构）和顶部账号栏。

**最稳妥做法**：在一个**独立测试 vault** 里用 `examples/topic.md` 和 `examples/review.md` 的虚构数据截图，避免触碰生产 vault。

## 嵌入后

截图放入 `docs/screenshots/` 后无需改代码——README 用相对路径引用，push 后 GitHub 会自动渲染。

## 真实 Vault 验证

截图仅作展示用途。真实 Obsidian 视觉、交互、网络抓取等仍需按 [installation.md 的独立 Vault 验证清单](installation.md#独立-vault-验证) 手动验证。
